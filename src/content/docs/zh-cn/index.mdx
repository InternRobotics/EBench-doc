  <p class="doc-intro">
    EBench（Embodied Benchmark）是一个基于 NVIDIA Isaac Sim 的室内 VLA 操作基准，涵盖长程任务、灵巧操作以及移动操作，旨在公平高效地比较模型在更具挑战性的任务上的能力。
  </p>

  <p class="doc-intro">
    EBench 采用 client–server 架构。Server 端运行 Isaac Sim 并以黑盒的方式启动仿真环境；Client 端是一个依赖极少的轻量包，可以直接安装到模型所在的环境中，通过 import 通信 client 与 Server 交互，避免依赖冲突。
  </p>
</div>

## Benchmark 范围

任务族：

- Pick-and-Place
- Long-Horizon
- Dexterous & Precise

泛化维度：

- 未见物体（unseen objects）
- 未见背景（unseen backgrounds）
- 未见指令（unseen instructions）
- 组合未见设置（`object + background + instruction`）

## 当前可提交的任务设置

- `ebench/mobile_manip/<split>`
- `ebench/table_top_manip/<split>`
- `ebench/generalist/<split>`

支持的 split：`val_train`、`val_unseen`、`test`

```bash
gmp submit ebench/mobile_manip/test
gmp submit ebench/table_top_manip/val_unseen
gmp submit ebench/generalist/val_train
```

## 快速开始

1. 克隆仓库并配置 Isaac Sim server 环境。
2. 在模型环境中安装 `genmanip-client`。
3. 下载资产：`huggingface-cli download InternRobotics/EBench-Assets --local-dir saved --repo-type dataset`。
4. 启动评测服务 `ray_eval_server.py`。
5. 用 `gmp submit` 提交任务，结果在 `saved/eval_results/` 中查看。

## 架构设计

EBench 将仿真和模型推理分离：

- **Server** — 运行 Isaac Sim 和评测逻辑，重依赖（Isaac Sim、cuRobo 等）集中在此。
- **Client** — 一个极轻量的 Python 包（`genmanip-client`），依赖极少。安装到模型环境后直接 `import` 通信 client 即可与 Server 交互。

这样模型环境保持干净，不会出现 Isaac Sim 和训练/推理栈之间的依赖冲突。
