  <p class="doc-intro">
    EBench (Embodied Benchmark) is an indoor VLA manipulation benchmark built on NVIDIA Isaac Sim.
    It covers long-horizon tasks, dexterous manipulation, and mobile manipulation, enabling fair and
    efficient comparison of model capabilities on challenging embodied tasks.
  </p>

  <p class="doc-intro">
    EBench uses a client–server architecture. The server runs Isaac Sim and exposes the simulation
    environment as a black box. A minimal client package, with very few dependencies, can be installed
    into the model's own environment and imported directly to communicate with the server — avoiding
    dependency conflicts entirely.
  </p>
</div>

## Benchmark scope

Task families:

- Pick-and-Place
- Long-Horizon
- Dexterous & Precise

Generalization dimensions:

- unseen objects
- unseen backgrounds
- unseen instructions
- combined unseen setting (`object + background + instruction`)

## Current task settings

- `ebench/mobile_manip/<split>`
- `ebench/table_top_manip/<split>`
- `ebench/generalist/<split>`

Supported splits: `val_train`, `val_unseen`, `test`

```bash
gmp submit ebench/mobile_manip/test
gmp submit ebench/table_top_manip/val_unseen
gmp submit ebench/generalist/val_train
```

## Quick start

1. Clone the repository and set up the Isaac Sim server environment.
2. Install the `genmanip-client` package into your model environment.
3. Download assets: `huggingface-cli download InternRobotics/EBench-Assets --local-dir saved --repo-type dataset`.
4. Start the evaluation server with `ray_eval_server.py`.
5. Submit a benchmark job with `gmp submit` and inspect results under `saved/eval_results/`.

## Architecture

EBench separates simulation from model inference:

- **Server** — runs Isaac Sim and the benchmark logic. Heavy dependencies (Isaac Sim, cuRobo, etc.) live here.
- **Client** — a minimal Python package (`genmanip-client`) with very few dependencies. Install it alongside your model, then `import` the communication client to interact with the server.

This design keeps your model environment clean and avoids dependency conflicts between Isaac Sim and your training/inference stack.
