  <p class="doc-intro">
    EBench is an Isaac Sim based embodied manipulation benchmark that evaluates multiple
    capability dimensions together: Pick-and-Place, Long-Horizon tasks, and Dexterous &amp; Precise
    manipulation. The supported usage path is built on GenManip&apos;s client-server workflow.
  </p>

  <p class="doc-intro">
    EBench uses a client–server architecture. The server runs Isaac Sim and exposes the simulation
    environment as a black box. A minimal client package, with very few dependencies, can be installed
    into the model's own environment and imported directly to communicate with the server — avoiding
    dependency conflicts entirely.
  </p>
</div>

## Repository roles

- `EBench`: benchmark data, trajectories, and packaged assets.
- `GenManip-Sim`: the evaluation server, benchmark task configs, and the project root that expects `saved/`.
- `genmanip-client`: provides `gmp` CLI and the `EvalClient` API for model-side integration.

## Working directories

- Run `ray_eval_server.py` from the `GenManip-Sim/` repository root.
- Install and run `gmp` from your client environment after installing `genmanip-client`.
- Keep benchmark data available under `GenManip-Sim/saved/`, either by placing files there directly or symlinking `EBench/` to that path.

## Benchmark scope

Task families:

- Pick-and-Place
- Long-Horizon
- Dexterous & Precise

Generalization dimensions in current settings:

- unseen objects
- unseen backgrounds
- unseen instructions
- combined unseen setting (`object + background + instruction`)

## Current task settings (for submit)

- `ebench/mobile_manip/<split>`
- `ebench/table_top_manip/<split>`
- `ebench/generalist/<split>`

Supported splits:

- `val_train`
- `val_unseen`
- `test`

Examples:

```bash
gmp submit ebench/mobile_manip/test
gmp submit ebench/table_top_manip/val_unseen
gmp submit ebench/generalist/val_train
```

## End-to-end flow

1. Prepare environment and install server/client dependencies
2. Pull EBench data and map it to `saved/`
3. Start `ray_eval_server.py`
4. Submit one or more benchmark configs with `gmp submit`
5. Run client workers (`gmp eval` or custom EvalClient loop)
6. Track progress using `gmp status` and read outputs under `saved/eval_results/`
